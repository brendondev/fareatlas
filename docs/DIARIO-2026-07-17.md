## 1. Integração Seats.aero (etapa 1 do GUIA-APIS)

### Objetivo
Conectar a Partner API do Seats.aero para buscar disponibilidade de assentos prêmio (Qantas, Velocity e programas parceiros), sem expor a API key no frontend.

### O que foi criado

| Peça | Caminho |
| --- | --- |
| Client HTTP | `src/lib/seats-aero/client.ts` |
| Tipos | `src/lib/seats-aero/types.ts` |
| Mapeamento UI | `src/lib/seats-aero/map.ts` |
| Labels de programas | `src/lib/seats-aero/programs.ts` |
| Barrel export | `src/lib/seats-aero/index.ts` |
| Busca | `GET /api/awards/search` |
| Detalhe de voos | `GET /api/awards/[id]/trips` |
| UI de busca | `src/components/award-search.tsx` |

### Comportamento da API

**Busca cacheada Seats.aero**

```
GET /api/awards/search?origin=SYD&destination=HND&start_date=YYYY-MM-DD&end_date=YYYY-MM-DD
```

Parâmetros suportados (entre outros): `sources`, `cabins`, `carriers`, `only_direct`, `take`, `order_by`, `include_trips`.

**Detalhe de itinerários**

```
GET /api/awards/{availabilityId}/trips
```

Retorna segmentos, milhas, taxas, assentos e `booking_links` oficiais.

### Autenticação Seats.aero

- Header: `Partner-Authorization`
- Env: `AWARD_AVAILABILITY_API_KEY` (alias: `SEATS_AERO_API_KEY`)
- Tokens OAuth `seats:…` recebem prefixo `Bearer` automaticamente

### Programas AU-focus (default)

`qantas`, `velocity`, `aeroplan`, `united`, `flyingblue`, `singapore`, `emirates`, `qatar`, `virginatlantic`, `american`, `alaska`, `delta`.

### Documentação de referência

- Getting started: https://developers.seats.aero/reference/getting-started-p  
- Cached Search: https://developers.seats.aero/reference/cached-search  
- Get Trips: https://developers.seats.aero/reference/get-trips  

**Nota comercial:** uso comercial da Partner API exige autorização escrita da Seats.aero.

### Scripts npm

| Script | Função |
| --- | --- |
| `npm run dev` | Servidor local |

- Watches respondem `source: "fallback"` sem persistir
- Cache Seats.aero fica desligado (chama API direto)

## 4. Cache diário Seats.aero (economia de cota)

### Problema
Plano Pro/Partner costuma ter limite baixo (ex.: **~1000 chamadas/dia**). Buscas repetidas queimam a cota.

### Solução (implementada em 17/07/2026)

```
Usuário pesquisa SYD→HND (mesmos filtros)
        │
        ▼
  Hash(query + dayKey Sydney)
        │
   ┌────┴────┐
   │  HIT?   │
   └────┬────┘
    sim │     não
        │      │
        ▼      ▼
     Neon   Seats.aero API
     JSON      │
               ▼
            Grava Neon
               │
               ▼
          Resposta UI
```

### Detalhes técnicos

| Aspecto | Implementação |
| --- | --- |
| Chave | SHA-256 de origem, destino, datas, sources, cabins, carriers, flags, take/skip/cursor + `dayKey` |
| Dia | Calendário **Australia/Sydney** (`en-CA` → `YYYY-MM-DD`) |
| Tabelas | `AwardSearchCache`, `AwardTripCache` |
| Hit | Incrementa `hitCount`, retorna `cache.source = "cache"` |
| Miss | Chama API, `upsert` no Neon, `cache.source = "api"` |
| Bypass | `?refresh=1` força API e regrava |
| Prune lazy | Ao gravar, apaga `dayKey` diferente de hoje |
| Purge diário | Cron Vercel limpa tabelas inteiras |

### Endpoints

| Método | Path | Função |
| --- | --- | --- |
| GET | `/api/awards/search` | Busca com cache |
| GET | `/api/awards/[id]/trips` | Trips com cache |
| GET/POST | `/api/cron/award-cache` | Manutenção |
| | `?mode=prune` | Remove dias antigos (default) |
| | `?mode=purge` | **Limpa tudo** (cron noturno) |

### Cron Vercel

Arquivo: `vercel.json`

```json
{
  "crons": [
    {
      "path": "/api/cron/award-cache?mode=purge",
      "schedule": "5 14 * * *"
    }
  ]
}
```

- **14:05 UTC** ≈ **00:05 AEST** (Sydney, horário padrão)
- Proteger com `CRON_SECRET` (`Authorization: Bearer …` ou `?secret=`)

### UI

A tabela de resultados em `/flights` mostra se a resposta veio:

- do **Neon cache**, ou  
- **fresh da Seats.aero** (e se foi salva)

## 8. Como rodar o estado de 17/07/2026

```bash
cd fareatlas
npm install
cp .env.example .env.local
# Preencher:
#   DATABASE_URL, DIRECT_URL   (Neon)
#   AWARD_AVAILABILITY_API_KEY (Seats.aero)
#   CRON_SECRET                (opcional local; recomendado na Vercel)

npx prisma db push
npm run db:seed
npm run dev
```

Abrir:

- http://localhost:3000 — landing  
- http://localhost:3000/offers — ofertas  
- http://localhost:3000/flights — busca award  
- http://localhost:3000/pricing — planos  

Teste de cache: repetir a mesma busca SYD→HND no mesmo dia; a segunda deve vir com `cache.source = "cache"`.

## 9. Decisões de produto tomadas nesta sessão

1. **Pasta correta:** trabalho no repo `fareatlas` (não no `nuvaryn-care`).  
2. **Posicionamento:** Austrália first; 4 programas AU; pontos **e** cash.

5. **Prisma 5.22** pinado (evitar breaking changes do Prisma 7 nesta fase).  
6. **Sem Neon o app ainda sobe** com demos — bom para preview e first deploy.
